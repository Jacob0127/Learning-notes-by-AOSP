# 去掉测试模式中的某个测试项

以`谷歌key、teekey、nfc`举例

## 🎯 标准定位流程（修正版）

### 第一步：找到`AndroidManifest.xml`

```bash
# 1. 首先定位AndroidManifest.xml
cd /work/work1/xiaowei/1_mtk14.0_su/mssi_A14/vendor/mediatek/proprietary/packages/apps

# 找到工程模式的AndroidManifest.xml
find . -name "AndroidManifest.xml" | grep -i engineer

# 通常是这个路径：
ls -la EngineerCode/AndroidManifest.xml# 进入项目目录
cd /work/work1/xiaowei/1_mtk14.0_su/mssi_A14/vendor/mediatek/proprietary/packages/apps

# 1. 搜索关键词（最常用）
grep -r "nfc\|teekey\|googlekey" --include="*.java" --include="*.xml" .

# 2. 查看工程模式目录结构
ls -la EngineerCode/
ls -la EngineerCode/src/com/zte/engineer/

# 3. 查找AndroidManifest.xml
find . -name "AndroidManifest.xml" -type f | xargs grep -l "EngineerMode"
```

### **第二步：分析AndroidManifest.xml**

查看清单文件，找到关键信息：

```xml
# 查看主入口Activity
<!-- 1. 主入口（可能是通过特殊代码启动） -->
<activity android:name=".EngineerCode">
    <intent-filter>
        <action android:name="com.zte.engineer.action.TEST_LIST" />
    </intent-filter>
</activity>

<!-- 2. 另一个入口页面 -->
<activity android:name="EngineerTestMainActivity">
    <intent-filter>
        <action android:name="com.zte.engineer.action.ENGINEERCODE_TEST" />
    </intent-filter>
</activity>

<!-- 3. 具体测试Activity（需要找的） -->
<activity android:name="GooglekeyTest" ... />
<activity android:name="TeekeyTest" ... />
<activity android:name="NfcTest" ... />
```

### **第三步：根据Activity名找到对应类**

```bash
# 1. 找到Activity对应的Java文件
# GooglekeyTest → GooglekeyTest.java
# TeekeyTest → TeekeyTest.java  
# NfcTest → NfcTest.java

# 2. 在工程模式目录中查找
cd EngineerCode
find src -name "*Googlekey*.java" -o -name "*Teekey*.java" -o -name "*Nfc*.java"

# 或者更通用的搜索
find src -name "*.java" | xargs grep -l "class.*Test.*extends"
```

### **第四步：逆向找到配置源头**

现在需要找到这些测试是如何被组织到列表中的：

```bash
# 1. 查看测试Activity的包名和类名
head -20 src/com/zte/engineer/GooglekeyTest.java

# 2. 搜索在哪里引用了这些测试类
grep -r "GooglekeyTest\|TeekeyTest\|NfcTest" src/ --include="*.java"

# 3. 关键发现：都在MyApplication.java中被引用
grep -n "GooglekeyTest\|TeekeyTest\|NfcTest" src/com/zte/engineer/MyApplication.java
```

### **第五步：分析MyApplication.java结构**

```java
// 1. 测试项列表配置
public static ArrayList<Integer> KEYS_MANUAL_TEST = new ArrayList<>();
public static ArrayList<Integer> KEYS_AUTO_TEST = new ArrayList<>();

// 2. 在initKeys()方法中添加测试项
static {
    if (GOOGLE_KEY_TEST_ENABLED) {
        KEYS_MANUAL_TEST.add(R.string.tegooglekey);  // ← 这里
        KEYS_AUTO_TEST.add(R.string.tegooglekey);
    }
}

// 3. Intent匹配
private void initTestIntentsMatch(...) {
    case R.string.tegooglekey:
        intentList.add(newIntent(this, GooglekeyTest.class, isAutoTest));
        break;
}
```

## ✅ **总结的正确流程**

1. **AndroidManifest.xml** → 找到Activity注册
2. **strings.xml** → 找到资源ID ([R.string.xxx](https://r.string.xxx/))
3. **MyApplication.java** → 找到三处引用并注释
4. **可选**：注释strings.xml中的字符串
5. **可选**：注释AndroidManifest.xml中的Activity声明

**最简流程**：其实只需要修改`MyApplication.java`中的三处代码，其他都是辅助确认。
