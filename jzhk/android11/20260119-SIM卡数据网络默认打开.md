# 问题

为什么`/sprd/build/make/tools/buildinfo.sh`在这个路径的文件中添加`echo "ro.com.android.mobiledata=true"`就可以完成**SIM卡数据网络默认打开**的需求？

# `buildinfo.sh` 的作用

```bash
# 路径：/sprd/build/make/tools/buildinfo.sh
# 这个脚本在编译时被执行，生成 system/build.prop 文件
```

```
graph TD
    A[buildinfo.sh] --> B[生成 build.prop]
```

## **1. 编译阶段：属性注入**

```bash
# buildinfo.sh 执行流程
编译系统 → 执行 buildinfo.sh → 生成 system/build.prop → 打包进 system.img
              ↑
              你的这行代码被写入
```

## **2. 系统启动：属性加载**

```bash
# Android 启动时
init 进程 → 加载 /system/build.prop → 设置系统属性
                                ↓
                ro.com.android.mobiledata=true 生效
```

## **3. 运行时：影响数据网络决策**

```java
// 关键代码路径
DataEnabledSettings.isUserDataEnabled()
    ↓
TelephonyProperties.mobile_data()
    ↓
读取系统属性 ro.com.android.mobiledata
    ↓
返回 true (你设置的值)
```

# 💡 **总结回答**

**为什么在 `buildinfo.sh` 中添加 `echo "ro.com.android.mobiledata=true"` 就能完成需求：**

1. **编译固化**：`buildinfo.sh` 在编译时执行，将属性写入 `system/build.prop`，随系统固件永久存在
2. **标准接口**：`ro.com.android.mobiledata` 是 Android 标准属性，被 `TelephonyProperties.mobile_data()` 读取
3. **默认值控制**：这个属性作为 `isUserDataEnabled()` 的默认值，影响数据开关的初始状态
4. **分层设计**：Android 的数据网络控制采用"用户设置 > 系统默认"的分层设计，你的修改在系统默认层
5. **展讯验证**：这是展讯平台推荐的标准做法，经过大量项目验证可靠
