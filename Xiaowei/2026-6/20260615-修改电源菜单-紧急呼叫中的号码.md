# 怎么找到修改这里的

## 一、在源码中搜索对应字符串资源

使用`find+grep`命令查找对应的字符串资源

```bash
find SYS/ -name "strings.xml" -type f | xargs grep -rn "急救信息\|紧急电话号码"
```

![image-20260615203306693](C:\Users\admin38\AppData\Roaming\Typora\typora-user-images\image-20260615203306693.png)

在这些文件中找到对应的，这里唯一对应的文件就是：

```bash
SYS/packages/services/Telephony/res/values-zh-rCN/strings.xml
```

## 二、再根据该字符串资源找到调用它的Java或者kotlin代码

还是使用`find+grep`命令查找，不过我们这次可以差不多锁定路径了，应该就是在`SYS/packages/services/Telephony`这个路径里

```bash
find SYS/packages/services/Telephony -name "*.java" | xargs grep -rn "emergency_information_hint\|numerous_emergency_numbers_title"
```

![image-20260615204145460](C:\Users\admin38\AppData\Roaming\Typora\typora-user-images\image-20260615204145460.png)

只有一个匹配的选项：`SYS/packages/services/Telephony/src/com/android/phone/EmergencyDialer.java`

然后看到这个代码中的`onCreate`的方法，我们可以看到这里面调用了这个方法。

```
setupEmergencyShortcutsView();
```

根据方法名可以看到这个方法是设置紧急快捷方式按钮的。差不多就是这里，从这个方法中看到它调用了这个方法。

```
updateLocationAndEccInfo();
```

这个方法又调用了`setLocationInfo();`而这个方法中，没有调用其他方法，但是可以看到数据是由`mShortcutViewConfig.getCountryIso();`获取的，那继续追。

找到`ShortcutViewUtils.java`文件中的静态内部类`Config`的这个`getCountryIso()`方法。可以看到这个方法中获取的是`mPhoneInfo`，而这个`mPhoneInfo`在构造器中通过`refresh()`方法赋值的。最后调用了`pickPreferredPhone()`方法。

这个方法中，调用了`getPromotedEmergencyNumberLists()`方法。又由这个方法调用了`getEmergencyNumberList()`方法

随后找到这个路径下的`TelephonyManager.java`文件中的`getEmergencyNumberList()`方法

```bash
SYS/frameworks/base/telephony/java/android/telephony/TelephonyManager.java
```

而这个`getEmergencyNumberList()`方法中调用了`ITelephony`这个接口类中的`getEmergencyNumberList()`方法，我们找到这个`ITelephony`接口类，按寻常的`java/kotlin`文件去搜是搜不到的，因为这个文件是`.aidl`文件，这是约定好的，`I`开头的接口类几乎都是`.aidl`类，所以我们搜索到这个`ITelephony.aidl`文件，路径：

```bash
SYS/frameworks/base/telephony/java/com/android/internal/telephony/ITelephony.aidl
```

这个文件只是接口文件，我们需要找到对应的实现类才行，找到继承了`ITelephony.Stub`这个的类。最后找到了`PhoneInterfaceManager.java`这个文件

```
SYS/packages/services/Telephony/src/com/android/phone/PhoneInterfaceManager.java
```
