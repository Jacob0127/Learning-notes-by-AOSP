### 1.首先通过字符串资源找到调用的该字符串的Java类，具体方法就是使用`find`和`grep`命令组合搜索。

```bash
# 例如
find . -name "strings.xml" -type f | xargs grep "深色主题"
```

不过如果知道这个下拉状态栏实现跳转大概模块实在SystemUI就查找的更快。

```bash
# 例如
find SYS/frameworks/base/packages/SystemUI -name "strings.xml" -type f | xargs grep -rn "深色主题"
```

通过这个命令找到了对应字符串名称

```bash
<string name="quick_settings_ui_mode_night_label" msgid="1398928270610780470">"深色主题"</string>
```

再根据这个字符串名称找到对应的Java类

```bash
# 命令
grep -rn "quick_settings_ui_mode_night_label" SYS/frameworks/base/packages/SystemUI/src
```

这样就可以找到`UiModeNightTile.java`



### 2.从该方法为入口一直追，查看完整的调用链

#### `UiModeNightTile.java`

在这个`UiModeNightTile.java`中，有一个关键方法`getLongClickIntent()`。

```java
// UiModeNightTile.java
public class UiModeNightTile extends QSTileImpl<QSTile.BooleanState> implements
        ConfigurationController.ConfigurationListener,
        BatteryController.BatteryStateChangeCallback {
	...
	@Override
	public Intent getLongClickIntent() {
    	return new Intent(Settings.ACTION_DARK_THEME_SETTINGS);
	}
    ...
}
```

从这个方法名可以看出来，这个是长按的逻辑，但是这里只是返回了一个`Intent`对象，并没有启动`Activity`，那么，问题来了：哪个方法调用了它，获得了这个Intent对象，并做了什么呢？

所以现在，需要去找到调用者，通过`getLongClickIntent`。

```bash
# 命令
grep -rn "getLongClickIntent" SYS/frameworks/base/packages/SystemUI/src
```

然后可以看到结果中有这样一段，定位到`QSTileImpl.java`类

```bash
# 结果
SYS/frameworks/base/packages/SystemUI/src/com/android/systemui/qs/tileimpl/QSTileImpl.java:447:        mActivityStarter.postStartActivityDismissingKeyguard(getLongClickIntent(), 0,
```



#### `QSTileImpl.java`

这个类的`handleLongClick()`方法中，调用了`getLongClickIntent()`。

```java
// QSTileImpl.java

public abstract class QSTileImpl<TState extends State> implements QSTile, LifecycleOwner, Dumpable {
	// ...
    protected void handleLongClick(@Nullable Expandable expandable) {
      	// ...
        // 关键点！
        ActivityTransitionAnimator.Controller animationController =
                expandable != null ? expandable.activityTransitionController(
                        InteractionJankMonitor.CUJ_SHADE_APP_LAUNCH_FROM_QS_TILE) : null;
        mActivityStarter.postStartActivityDismissingKeyguard(getLongClickIntent(), 0,
                animationController);
    }
	//...
}
```

同理，继续找哪个方法调用`handleLongClick()`方法，可以看到，在同类中的`handleMessage()`方法中调用了该方法

```java
// QSTileImpl.java

@Override
public void handleMessage(Message msg) {
    String name = null;
    try {
        if (msg.what == ADD_CALLBACK) {
            // ...
        } else if (msg.what == LONG_CLICK) {
            name = "handleLongClick";
            mQSLogger.logHandleLongClick(mTileSpec, msg.arg1);
            handleLongClick((Expandable) msg.obj); // 在这里调用了
        } 
     // ...
    } catch (Trowable t) {
        // ...
    }
}
```

继续找，这个`handleMessage()`只是用来发送消息的，还需要找到是哪个方法发送了"LONG_CLICK"的消息。还是在这个类中，找到了发送"LONG_CLICK"消息的方法`longClick()`

```java
// QSTileImpl.java

@Override
public void longClick(@Nullable Expandable expandable) {
        // ...
        if (!mFalsingManager.isFalseLongTap(FalsingManager.LOW_PENALTY)) {
            mHandler.obtainMessage(H.LONG_CLICK, eventId, 0, expandable).sendToTarget(); // 在这里发送了LONG_CLICK消息
        }
    	// ...
    }
```

继续找谁调用了`longClick()`方法。最后能找到`QSTileViewImpl.kt`这个类。



#### `QSTileViewImpl.kt`

这个类中的`init()`方法调用了`longClick()`方法

```kotlin
// QSTileViewImpl.kt

override fun init(tile: QSTile) {
    // ...
    else {
        val expandable = Expandable.fromView(this)
        init(
            { _: View? -> tile.click(expandable) },
            { _: View? ->
             tile.longClick(expandable) // 这里调用了longClick()方法
             true
            },
        )
    }
}
```
