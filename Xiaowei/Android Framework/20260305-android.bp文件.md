# bp文件介绍

bp文件是为了替换mk文件，与mk文件不同，bp文件不能做分支、循环等流程控制，它是纯粹的配置文件，bp文件记录着模块信息，**每一个模块以模块类型开始，后面跟着模块的属性**，以名值对（`name: value`）表示，每个模块都**必须**要有一个`name`属性。

# 常见的编译类型

## android_app

- app，就是编写一个apk

## cc_binary

- 可执行文件

## cc_library_shared

- native动态库

## cc_library_static

- native静态库

## java_library

- java库

# 基本使用

如果要编写apk，必须先指定模块源码路径和编译后的模块名称

```makefile
android_app {
	// 编译后的模块名称
	name: "messaging",
	
	// 该模块指定的源码路径，相对于bp文件所在目录的路径
	src: ["src/**/*.java"]
}
```

> 这个`messagin`就是模块名称，当需要改了某个模块的代码时，看到有bp文件，可以看看这个模块名称，再使用单编模块命令即可单编模块

# 许可证声明

每个 Android.bp 文件应该包含许可证声明：

```makefile
package {
    default_applicable_licenses: ["Android-Apache-2.0"],
}
```

> 从Android 10 开始强制要求，可以继承父目录的声明

# 常用属性

```makefile
android_app {
    name: "MyApp",
    srcs: ["src/**/*.java"],
    resource_dirs: ["res"],           // 资源目录
    manifest: "AndroidManifest.xml",  // Manifest 文件路径
    deps: [                           // 依赖的其他模块
        "androidx.appcompat_appcompat",
    ],
    static_libs: ["mylib"],           // 静态链接库
    sdk_version: "current",           // SDK 版本
    platform_apis: true,              // 使用平台 API
}
```

# filegroup模块

用于**将一组文件打包成一个逻辑名称**，方便其他模块引用。它本身不产生编译输出，只是一个"文件集合"的别名。

## 作用说明

`filegroup` 的核心作用是：

1. **简化依赖声明**：多个模块需要相同的一组文件时，只需引用一个 filegroup
2. **路径复用**：避免重复写长长的文件路径列表
3. **条件编译**：可以根据不同配置包含不同文件

```makefile
filegroup {
    name: "launcher-src",           // 文件集合的名称
    srcs: [                          // 包含的文件列表
        "src/**/*.java",             // 递归所有 Java 文件
        "src/**/*.kt",               // 递归所有 Kotlin 文件
        "JzhkLauncher3/src/**/*.java", // 另一个目录的 Java 文件
    ],
}
```

## 如何使用

其他模块可以通过 `srcs` 属性引用这个 filegroup：

```javascript
// 方式1：直接作为 srcs（使用 : 前缀）
android_app {
    name: "Launcher3",
    srcs: [":launcher-src"],  // 引用上面的 filegroup
    // 还可以添加额外文件
    srcs: [
        ":launcher-src",
        "src/debug/**/*.java",
    ],
}

// 方式2：与 java_defaults 配合
java_defaults {
    name: "launcher-defaults",
    srcs: [":launcher-src"],
}

android_app {
    name: "QuickStep",
    defaults: ["launcher-defaults"],  // 间接获得这些源文件
}
```

## 引用语法说明

| 写法                   | 含义                                                     |
| :--------------------- | :------------------------------------------------------- |
| `":模块名"`            | 引用**当前目录**的模块（必须在同一个 Android.bp 文件中） |
| `"//路径:模块名"`      | 引用**其他目录**的模块                                   |
| `"相对路径/文件.java"` | 直接指定源文件                                           |



# optimize代码优化配置块

```javascript
optimize: {
    // 指定 ProGuard 规则文件
    proguard_flags_files: [":launcher-proguard-rules"],
    
    // 是否启用 ProGuard
    enabled: false,  // false = 禁用混淆/优化
},
```

- `proguard_flags_filesproguard_flags_files`:ProGuard 规则文件列表，定义哪些代码保留、哪些删除
