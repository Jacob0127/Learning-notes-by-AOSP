# `mmm`命令的作用

**问题**：使用make命令编译了源码之后，修改了源代码中的某个模块或者添加了一个模块，是否还需要再使用make命令编译一次呢？

答案当然是不用的，我们可以使用`mmm`命令来单独编译某一个模块，并提供了`make snod`命令来纯粹执行打包Android系统镜像文件`system.img`的操作。使用`mmm`命令可以提高编译速度。

# 使用`mmm`命令的前提

默认情况下，`mmm`是不可使用的，我们需要再当前终端中执行`build/envsetup.sh`脚本文件，才可以使用`mmm`命令。

```bash
:$ source ./build/envsetup.sh
```

# `mmm`的使用

```bash
# 语法：mmm + 应用的目录
:$ mmm ./packages/app/Launcher3
```

> 注意：这个应用的目录下必须存在一个编译脚本文件**Android.mk**

编译完成之后，还**需要重新打包Android系统镜像文件system.img**

# 重新打包Android系统镜像文件system.img

执行make snod命令，即可重新打包Android系统镜像文件system.img

```bash
# snod(System image NO Dependency 不检查依赖的系统镜像打包)
:$ make snod
```
