# 什么情况需要实现自定义注解驱动切点

需要对特定的业务逻辑进行统一处理，比如：权限校验、日志记录等

# 如何实现

# 1.添加依赖

在pom.xml文件中添加aop的依赖

```xml
<!-- aop -->
<dependency>
	<groupId>org.springframework.boot</groupId>
	<artifactId>spring-boot-starter-aop</artifactId>
</dependency>
```

# 2.在启动类上添加`@EnableAspectJAutoProxy`注解表示启动切面编程

```java
@SpringBootApplication
@EnableAspectJAutoProxy
public class SmartAiApplication {

    public static void main(String[] args) {
        SpringApplication.run(SmartAiApplication.class, args);
    }

}
```



# 3.自定义注解

首先，创建一个注解类，并给注解类添加上Java的元注解`@Target`和`@Rentention`

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface TestAnnotation {
	//具体实现
}
```

拿权限校验来举例，下面是我自定义注解类

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface AuthCheck {

    /**
     * 必须有某个角色
     */
    String mustRole() default "";
}
```



下面我来解释一下`@Target`和`@Retention`两个注解的作用

## `@Target`

用于指定注解可以作用在Java程序的哪些位置

### 基本语法

```java
单一用法：
@Target(ElementType.METHOD)
public @interface AuthCheck { // 具体实现 }

多作用：
@Target({ElementType.METHOD, ElementType.TYPE})
public @interface AuthCheck { // 具体实现 }
```

### `ElementType`枚举值

| `ElementType`     | 说明                 | 示例                             |
| :---------------- | :------------------- | :------------------------------- |
| `TYPE`            | 类、接口、枚举       | `@Controller`, `@Service`        |
| `FIELD`           | 字段（成员变量）     | `@Autowired`, `@Value`           |
| `METHOD`          | 方法                 | `@GetMapping`, `@Override`       |
| `PARAMETER`       | 方法参数             | `@RequestParam`, `@PathVariable` |
| `CONSTRUCTOR`     | 构造方法             | -                                |
| `LOCAL_VARIABLE`  | 局部变量             | -                                |
| `ANNOTATION_TYPE` | 注解类型             | `@Target`, `@Retention`          |
| `PACKAGE`         | 包                   | -                                |
| `TYPE_PARAMETER`  | 类型参数（泛型）     | Java 8+                          |
| `TYPE_USE`        | 类型使用（任何类型） | Java 8+                          |
| `MODULE`          | 模块                 | Java 9+                          |



### 注意：

1. **明确注解用途**：根据注解的用途选择合适的 `@Target`
2. **保持单一职责**：一个注解最好只做一件事
3. **合理使用组合**：可以使用多个 `@Target`，但要确保合理
4. **考虑扩展性**：设计注解时考虑未来可能的扩展
5. **文档注释**：为自定义注解添加详细的JavaDoc

## `@Retention`

用于指定注解的生命周期（即注解在何时有效）

### 基本语法

```java
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {
    // 注解内容
}
```

### `RetentionPolicy` 枚举值

| RetentionPolicy | 生命周期   | 说明                              | 使用场景                   |
| :-------------- | :--------- | :-------------------------------- | :------------------------- |
| `SOURCE`        | 源码级别   | 仅在源码中存在，编译时丢弃        | 编译时检查、代码生成       |
| `CLASS`         | 字节码级别 | 保留到字节码文件，JVM运行时不可见 | 字节码处理、APT            |
| `RUNTIME`       | 运行时级别 | 保留到运行时，可通过反射获取      | Spring注解、自定义业务注解 |

### 三种保留策略

#### `SOURCE`级别

##### 特点

- 仅在源码中存在
- 编译后不保留到.class文件
- 主要用于编译时检查

##### 示例：

```java
// 自定义源码级别注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.SOURCE)
public @interface NotNullParameters {
}

// 使用
public class Calculator {
    
    @NotNullParameters
    public int divide(int a, int b) {
        // 编译时检查参数是否为空
        return a / b;
    }
}
```



#### `CLASS`级别

##### 特点

- 编译后保留到.class文件
- JVM加载类时不会加载到内存
- 可用于字节码增强、代码生成

##### 示例

```java
// 用于字节码增强的注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.CLASS)
public @interface ProfileMethod {
    // 标记需要性能监控的方法
}

// 字节码处理工具会在编译后修改标记的方法
```



#### `RUNTIME`级别（最常用）

##### 特点

- 编译后保留到.class文件
- JVM加载类时会加载到内存
- **可通过反射获取**
- Spring、MyBatis等框架主要使用这种

##### 示例

```java
import java.lang.annotation.*;

// 自定义权限注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RequiresRole {
    String[] value();
    Logical logical() default Logical.AND;
}

// 使用
@RestController
public class UserController {
    
    @RequiresRole({"ADMIN", "MANAGER"})
    @GetMapping("/admin/users")
    public List<User> getAdminUsers() {
        // 需要ADMIN和MANAGER角色
    }
    
    @RequiresRole(value = {"USER", "GUEST"}, logical = Logical.OR)
    @GetMapping("/public/profile")
    public UserProfile getPublicProfile() {
        // 需要USER或GUEST角色之一
    }
}
```

### 如何选择使用那种`RetentionPolicy`枚举值

##### 只需要编译检查时 -> `SOURCE`：

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.SOURCE)
public @interface Nullable {
    // 编译器检查参数是否可为null
}
```

##### 需要字节码处理 -> `CLASS`

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.CLASS)
public @interface GenerateBuilder {
    // 编译时生成Builder类
}
```

需要运行时处理 ->`RUNTIME`

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Cacheable {
    // 运行时根据注解进行缓存
}
```

# 4.定义一个切面类来匹配注解

```java
@Component
@Aspect
public class TestAOP {
    
    @Around("@annotation(authCheck)")
    public Object testAOP(ProceedingJoinPoint joinPoint, AuthCheck authCheck) {
        // 具体实现
        return joinPoint.proceed();// 放行
    }
}
```



下面是我的写的用来处理权限管理的切面类

```java
@Component
@Aspect
public class AuthInterceptor {

    @Resource
    private UserService userService;

    @Around("@annotation(authCheck)")
    public Object doIntercept(ProceedingJoinPoint joinPoint, AuthCheck authCheck) throws Throwable {
        String mustRole = authCheck.mustRole();
        /**
         * RequestContextHolder
         * 作用：Spring提供的工具类，用于存储和访问当前线程的请求上下文信息
         * 功能：提供对请求属性的访问，即使在非Web线程中也能获取到请求相关信息
         *
         * RequestAttributes
         * 作用：封装了请求范围内的属性信息
         * 功能：提供对请求参数、会话属性等的统一访问接口
         *
         * ServletRequestAttributes
         * 作用：RequestAttributes的具体实现类，专门针对Servlet环境
         * 功能：将原始的HttpServletRequest和HttpServletResponse包装成RequestAttributes
         *
         * HttpServletRequest
         * 作用：Java Servlet API中的核心接口
         * 功能：封装HTTP请求的所有信息，包括请求头、参数、会话等
         *
         * 代码执行流程:
         * 1.通过RequestContextHolder获取当前线程的请求属性
         * 2.将RequestAttributes向下转型为ServletRequestAttributes
         * 3.从ServletRequestAttributes中获取原始的HttpServletRequest对象
         */
        // 获取当前登录用户
        RequestAttributes requestAttributes = RequestContextHolder.currentRequestAttributes();
        HttpServletRequest request = ((ServletRequestAttributes) requestAttributes).getRequest();

        String token = request.getHeader("token");
        if (JwtConfig.validateToken(token)) {
            throw new BusinessException(ErrorCode.OPERATION_ERROR, "token已失效请重新登录");
        }
        String userRole = JwtConfig.getUserTypeFromToken(token);
        if (userRole.equals(mustRole) || userRole.equals("root") || userRole.equals("admin")) {
            throw new BusinessException(ErrorCode.NO_AUTH_ERROR, "无权限");
        }
        return joinPoint.proceed();
    }
}
```
