# 一、修改的核心逻辑是：系统如何识别手机系统

Android 系统中，手机的“型号”信息主要通过以下几种方式获取和展示：

1. **系统属性（System Properties）**
   - 例如：`ro.build.display.id`、`ro.product.model`、`persist.sys.modelname` 等
   - 很多系统应用和设置界面会读取这些属性来显示型号
   - 修改方法：在 `buildinfo.sh` 或 `build.prop` 中设置
2. **蓝牙设备名称**
   - 蓝牙对外广播的设备名称通常与型号相关
   - 修改方法：在蓝牙模块代码中设置默认名称
3. **MTP（媒体传输协议）设备信息**
   - 连接电脑时，MTP 设备会向电脑报告设备型号
   - 修改方法：在 MTP 服务器代码中设置
4. **系统 UI 文本动态替换（Hook）**
   - 有些应用（如安兔兔、CPU-Z）会通过其他方式读取设备信息
   - 为了在这些应用中也能显示目标型号，系统会在文本渲染时动态替换特定字符串

# 二、修改

## 1.`buildinfo.sh`（编译脚本）

```bash
- echo "ro.build.display.id=17 Pro Max-M-HORSE-V01-`date +%Y%m%d`"
+ echo "ro.build.display.id=A17-M-HORSE-V01-`date +%Y%m%d`"
```

- 作用：在编译时生成系统属性，影响“关于手机”中的版本显示
- 说明：这里的 `ro.build.display.id` 是系统版本字符串的一部分，通常会显示在“设置-关于手机”中

```bash
- echo "persist.sys.wifiname=17 Pro Max"
+ echo "persist.sys.wifiname=A17"
- echo "persist.sys.modelname=17 Pro Max"
+ echo "persist.sys.modelname=A17"
```

- 作用：设置 WiFi 名称和系统模型名称的系统属性
- 说明：`persist.sys.modelname` 会被很多系统组件读取，作为“设备型号”的默认值

## 2. `MtpServer.cpp`（MTP 协议服务）

```cpp
- property_get("persist.sys.modelname", prop_value, "17 Pro Max");
+ property_get("persist.sys.modelname", prop_value, "A17");
```

- 作用：当手机通过 USB 连接电脑时，MTP 服务会向电脑报告设备型号
- 说明：这里读取的是上面设置的 `persist.sys.modelname` 属性，确保电脑端看到的也是“A17”

## 3. `TextView.java`（系统文本视图类）

这是修改最多的地方，也是最“黑科技”的部分：
**为什么要在 TextView 里改？**
因为很多第三方应用（如安兔兔、CPU-Z、Google Photos 等）会通过非标准方式读取设备信息，或者系统在某些界面显示时直接调用了硬编码的字符串。为了在这些场景下也显示“A17”，系统会**在文本绘制时动态替换**。

例如：

```java
if (findStr.contains("Model_6")) {
    text = findStr.replace("Model_6","A17");
}
```

- 作用：当 TextView 显示的文本中包含“Model_6”时，自动替换为“A17”
- 说明：这是一种“运行时字符串替换”机制，用于覆盖那些不从系统属性读取型号的应用

## 4. `DatabaseHelper.java`（设置数据库）

```java
- return "17 Pro Max";
+ return "A17";
```

- 作用：设置蓝牙设备名称的默认值（在“设置-蓝牙-设备名称”中可见）
- 说明：这个值也会被蓝牙模块读取并用于广播

## 5. `btif_dm.cc`（蓝牙接口层）

```cpp
- strncpy(btif_default_local_name, "17 Pro Max", max_len);
+ strncpy(btif_default_local_name, "A17", max_len);
```

- 作用：设置蓝牙默认本地设备名称
- 说明：当蓝牙开启时，其他设备搜索到的蓝牙名称就是这个值
