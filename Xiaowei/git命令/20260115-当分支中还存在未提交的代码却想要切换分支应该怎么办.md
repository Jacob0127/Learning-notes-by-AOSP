# 问题

在开发过程中，在一个分支开发新的功能，还没开发完毕，做到一半时有反馈需要处理紧急bug，但是新功能开发了一半又不想提交。

# 使用`git stash`命令

它会保存当前工作进度，会把暂存区和工作区的改动保存到一个未完结变更的堆栈中；执行完这个命令后，在运行 `git status` 命令，就会发现当前是一个干净的工作区，没有任何改动。

## 语法

```bash
git stash (save "注释信息") #括号中的可以不要
```

之后就可以使用`git stash list`来查看所有隐藏的代码

![image-20260115154210621](C:\Users\admin38\AppData\Roaming\Typora\typora-user-images\image-20260115154210621.png)

`stash@{0}`这个`@{}`中的数字就代表`id`，之后可以用于恢复隐藏

# 如何恢复隐藏

使用`git stash apply id`就可以将那一次的隐藏恢复过来

![image-20260115154452274](C:\Users\admin38\AppData\Roaming\Typora\typora-user-images\image-20260115154452274.png)



还有一个问题就是虽然使用`git stash apply 加上id`恢复了隐藏，但是这个暂存栈中还是存在已经恢复或者不需要的历史，那么如何删除暂存栈中的历史呢？

# 如何删除暂存栈中的历史

使用`git stash dorp id`即可
