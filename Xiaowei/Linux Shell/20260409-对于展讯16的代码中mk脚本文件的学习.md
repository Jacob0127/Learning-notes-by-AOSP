# `if [ "${BASH_SOURCE[0]}" = "${0}" ]; then`

```bash
if [ "${BASH_SOURCE[0]}" = "${0}" ]; then
```



| 变量                | 含义                                               |
| :------------------ | :------------------------------------------------- |
| `${BASH_SOURCE[0]}` | 当前脚本的**实际路径**（无论怎么执行都能正确获取） |
| `${0}`              | **当前执行的脚本名称**                             |

**判断逻辑**：

- 如果两者相等 → 说明用户是通过 `./mk` 或 `bash mk` 的方式**直接执行**脚本
- 如果两者不等 → 说明用户是通过 `source mk` 或 `. mk` 的方式**在当前 shell 中加载**脚本

## Shell知识点-脚本执行方式

虽然`./`、`bash`、`source`、`.`、`sh`都可以执行脚本文件，但它们在**执行环境、权限要求**和**变量作用域**上有着根本性的区别。核心区别在于：是**在当前Shell进程中执行**，还是**创建子Shell进程执行**。

它们的核心差异整理如下：

| 命令            | 执行环境            | 权限要求                 | 变量影响（在脚本内设置变量）              |
| :-------------- | :------------------ | :----------------------- | :---------------------------------------- |
| `source` 或 `.` | **当前Shell进程**   | 仅需**读**权限           | **会保留**在当前Shell中，直接影响当前环境 |
| `bash` 或 `sh`  | **新建子Shell进程** | 仅需**读**权限           | **不会保留**，子进程的变量随其退出而销毁  |
| `./`            | **新建子Shell进程** | 需要**读**和**执行**权限 | **不会保留**，子进程的变量随其退出而销毁  |



# `[[ ! -e ./.git ]] && [[ ! -e ${0##*/} ]] && echo 'Error: not in complie dir' && return 1`

```bash
[[ ! -e ./.git ]] && [[ ! -e ${0##*/} ]] && echo 'Error: not in complie dir' && return 1
```

## 1. 条件1：`[[ ! -e ./.git ]]`

| 部分     | 含义                                           |
| :------- | :--------------------------------------------- |
| `./.git` | 当前目录下的 `.git` 文件夹（Git 版本控制目录） |
| `-e`     | 检查文件/目录是否存在                          |
| `!`      | 取反，表示"不存在"                             |

**含义**：检查当前目录**是否不是** Git 仓库的根目录。

## 2. 条件2：`[[ ! -e ${0##*/} ]]`

| 部分       | 含义                                                       |
| :--------- | :--------------------------------------------------------- |
| `${0}`     | 脚本的名称（如 `./mk` 或 `mk`）                            |
| `${0##*/}` | 去掉所有前缀路径，只保留文件名。例如 `./scripts/mk` → `mk` |
| `-e`       | 检查文件是否存在                                           |

**含义**：检查**脚本自身**是否不存在于当前目录。

## Shell知识点

### -e

判断文件是否存在

### ${0##*/}

去掉所有前缀路径，只保留文件名。例如 `./scripts/mk` → `mk`

### ${0}

脚本的名称（如 `./mk` 或 `mk`）

| 执行方式      | `${0}` 的值                       |
| :------------ | :-------------------------------- |
| `./mk`        | `./mk`                            |
| `bash mk`     | `mk`                              |
| `/path/to/mk` | `/path/to/mk`                     |
| `. mk`        | `bash` 或 `-bash`（取决于 shell） |
| `source mk`   | `bash` 或 `-bash`                 |

# `jzhk_main $@`

```bash
jzhk_main $@
```

| 部分        | 含义                                   |
| :---------- | :------------------------------------- |
| `jzhk_main` | 函数名（应该在之前或后续被定义）       |
| `$@`        | 所有位置参数的列表（保持原样，带引号） |

## `$@` 详解

`$@` 代表脚本接收到的**所有参数**，每个参数都作为独立的单词。

```bash
# 假设你执行
. mk new build debug

# 那么
$@ = "new" "build" "debug"  # 三个独立的参数
```

# `local original_stty=$(stty -g)`

```bash
local original_stty=$(stty -g)
```

| 部分            | 含义                                                  |
| :-------------- | :---------------------------------------------------- |
| `local`         | 声明一个局部变量（仅在函数内有效）                    |
| `original_stty` | 变量名，用于保存原始状态                              |
| `$(stty -g)`    | 执行 `stty -g` 命令，获取当前终端设置的**字符串表示** |
| `stty -g`       | 以可恢复的格式输出当前终端设置                        |

# `if [[ $# -ne 2 ]]; then`

```bash
if [[ $# -ne 2 ]]; then
```

| 部分  | 含义                      |
| :---- | :------------------------ |
| `$#`  | 传递给脚本/函数的参数个数 |

# `  [ -z $catch ] && echo "Error: source project \"$jzhk_source_project\" can't match \"$jzhk_projects\"" && return 1`

```bash
    [ -z $catch ] && echo "Error: source project \"$jzhk_source_project\" can't match \"$jzhk_projects\"" && return 1
```

| 部分        | 含义                                        |
| :---------- | :------------------------------------------ |
| `-z $catch` | 检查 `catch` 变量是否为空（即是否匹配成功） |
| `-z`        | zero，字符串长度为0时为真                   |
| `return 1`  | 匹配失败，返回错误码1                       |

# ``jzhk_user_select "`find $(cd $jzhk_mk_path && pwd) -name '*.mk' | sort`" jzhk_project_mk``

| 部分                    | 含义                                               |
